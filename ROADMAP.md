# Roadmap

This project already has a working Transformer implementation, synthetic demo runs, and public-data validation.
The next steps should make it easier for others to contribute and easier to present as a product.

## Near-term

- add a small web demo
- add an inference API
- add more tests around evaluation and generation
- add more baselines and ablations
- make benchmark outputs easier to compare

## Mid-term

- improve model packaging for reuse
- add model/version metadata
- add simple monitoring hooks
- add a public contribution guide and issue templates
- add a lightweight demo deployment

## Longer-term

- support fine-tuning workflows
- add experiment tracking
- add load testing for the inference path
- add optional hosted demo support
- expand the public benchmark suite

## Contribution-friendly goals

Good community contributions should be small, well-scoped, and easy to review.
Examples:

- fix a bug
- improve documentation
- add a test
- add a baseline
- improve a result report
- make a CLI friendlier
