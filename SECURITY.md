# Security Policy

## Supported versions

This project currently tracks the latest `main` branch.

## Reporting a vulnerability

If you find a security issue, please open a private security advisory or contact the maintainer directly instead of posting details publicly in an issue.

Include:

- a short description of the issue
- affected files or commands
- steps to reproduce
- any suggested fix or mitigation

## Safe contribution guidance

- do not commit secrets or API keys
- do not add insecure default settings
- prefer minimal permissions for any automation
- keep dependencies up to date
